# saucedemo-qa-automation
This project establishes a comprehensive, high-impact quality assurance framework for the SauceDemo E-Commerce Platform. It integrates a business-centric approach with advanced test automation using Python and Playwright, ensuring a robust and reliable online shopping experience.


# SauceDemo E‑Commerce Platform  
## BA → QA → Automation Portfolio Project

This repository demonstrates how a real-world e‑commerce platform can be delivered
using structured Business Analysis, Quality Engineering, and Test Automation practices.

## What This Project Shows
- Business‑driven requirements (BRD & FRD)
- Clear traceability from requirements to tests (RTM)
- Risk‑based test design
- Automation focused on high‑value user journeys
- Agile, sprint‑based delivery approach

## System Under Test
https://www.saucedemo.com  
(A public demo application by Sauce Labs)

## Delivery Approach
The project follows an Agile, sprint‑by‑sprint approach:

- Sprint 0: Vision, repository setup, foundations 
- Sprint 1–3: Business & functional requirements
- Sprint 4: Test strategy and manual testing
- Sprint 5–7: Automation framework and implementation
- Sprint 8+: CI integration and reporting

## Repository Structure
- /docs → Business, requirements, and testing artifacts  
- /automation → Playwright + Python framework  
- /ci → Pipeline and quality gates  

## Key Outcome
This project focuses on **how quality software is delivered**, not just how tests are written.
